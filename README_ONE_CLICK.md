# Запуск одним кликом — START_WINDOWS.bat

Самый простой способ собрать и запустить **OzonFbsLabelPrinter** на Windows.

---

## Быстрый старт

1. Распакуйте архив в папку без кириллицы и пробелов.
   - Хорошо: `C:\Projects\ozon_fbs_label_app`
   - Плохо: `C:\Мои документы\приложение ozon`
2. Дважды щёлкните по файлу **`START_WINDOWS.bat`**.
3. Дождитесь завершения (первый запуск занимает 3–7 минут — скачиваются зависимости).
4. Когда откроется папка с приложением и появится вопрос `Запустить сейчас? [Y/N]` — нажмите `Y`.

Всё. Больше ничего делать не нужно.

---

## Что делает START_WINDOWS.bat

Файл выполняет все шаги автоматически:

| Шаг | Действие |
|-----|----------|
| 0 | Ищет Python (`py -3.12` → `py -3.11` → `py -3.10` → `py -3` → `python`) и проверяет совместимость с PySide6 |
| 1 | Создаёт виртуальное окружение `.venv` |
| 2 | Устанавливает зависимости (`requirements.txt`, `pyinstaller`, `pywin32`, `pytest`) |
| 3 | Запускает быструю проверку кода и тесты |
| 4 | Собирает `OzonFbsLabelPrinter.exe` через PyInstaller |
| 5 | Открывает папку с результатом |
| 6 | Предлагает запустить приложение |

При повторном запуске шаги 1–2 выполняются быстро (`.venv` уже готово).

---

## Где находится собранный EXE

После успешной сборки файл находится здесь:

```
ozon_fbs_label_app\
└── dist\
    └── OzonFbsLabelPrinter\          ← скопируйте всю папку, если нужно перенести
        ├── OzonFbsLabelPrinter.exe   ← запускаемый файл
        ├── templates\                 ← шаблоны этикеток
        ├── _internal\                 ← Python и все зависимости
        └── ...
```

> **Важно:** при переносе копируйте **всю папку** `OzonFbsLabelPrinter\`, а не только `.exe`.

## Требования к Python для PySide6

PySide6 (GUI-библиотека приложения) поддерживает **не все** версии Python.

| Требование | Подробности |
|---|---|
| Операционная система | Windows 10 или Windows 11 (64-bit) |
| Python | **3.10, 3.11, 3.12 или 3.13** — рекомендуется **3.11 или 3.12** |
| Разрядность | Только **64-bit** (x64) |
| Дистрибутив | **CPython** с [python.org](https://www.python.org/) |
| Не поддерживается | 32-bit Python, PyPy, Microsoft Store Python, Python < 3.10 или ≥ 3.14 |

Скрипт `START_WINDOWS.bat` автоматически проверяет совместимость перед установкой.

## Если Python не найден

Появится сообщение:
```
ERROR: Python was not found on this computer.
```

**Что делать:**

1. Откройте [https://www.python.org/downloads/windows/](https://www.python.org/downloads/windows/)
2. Скачайте **Python 3.11 или 3.12 (64-bit)**.
3. Запустите установщик.
4. **Обязательно** отметьте галочку **«Add python.exe to PATH»** на первом экране.
5. Нажмите «Install Now».
6. После установки закройте все окна командной строки и запустите `START_WINDOWS.bat` снова.

---

## Ошибка «No matching distribution found for PySide6»

Если вы видите такую ошибку при установке:
```
ERROR: Could not find a version that satisfies the requirement PySide6>=6.5.0 (from versions: none)
```

**Причина:** Python несовместим с PySide6. Чаще всего это 32-bit Python, слишком старая версия или Python из Microsoft Store.

**Как исправить:**
1. Удалите папку `.venv` в папке проекта.
2. Установите **Python 3.11 или 3.12 (64-bit)** с [python.org](https://www.python.org/downloads/windows/).
3. При установке отметьте **«Add python.exe to PATH»**.
4. Запустите `START_WINDOWS.bat` снова.

Для диагностики запустите **`CHECK_PYTHON_WINDOWS.bat`** — он покажет все найденные Python и их совместимость с PySide6.

## Если SmartScreen или антивирус блокирует EXE

Это нормальное поведение для приложений без цифровой подписи.

### Windows SmartScreen

При первом запуске появится синее окно «Windows защитила ваш компьютер»:

1. Нажмите **«Подробнее»** (синяя ссылка внизу окна).
2. Нажмите **«Выполнить в любом случае»**.
3. Приложение запустится. Это предупреждение появляется только при первом запуске.

### Антивирус (Kaspersky, ESET, Windows Defender и др.)

1. Откройте настройки антивируса.
2. Добавьте папку `dist\OzonFbsLabelPrinter\` в **исключения** (белый список).
3. Если антивирус уже удалил файлы — восстановите из карантина и добавьте в исключения.
4. Запустите `START_WINDOWS.bat` снова — PyInstaller пересоберёт EXE.

## Если что-то пошло не так при сборке

| Симптом | Решение |
|---------|---------|
| Ошибка `No matching distribution found for PySide6` | Удалите `.venv`, установите Python 3.11/3.12 (64-bit), запустите снова. |
| Ошибка при установке зависимостей | Проверьте интернет-соединение. Попробуйте отключить VPN. |
| Ошибка сборки (PyInstaller) | Добавьте папку проекта в исключения антивируса |
| «Недостаточно прав» | Щёлкните ПКМ по `START_WINDOWS.bat` → «Запуск от имени администратора» |
| Конфликт зависимостей | Удалите папку `.venv` и запустите `START_WINDOWS.bat` снова |
| EXE запускается, но нет шаблонов | Убедитесь что скопировали **всю папку** `OzonFbsLabelPrinter\`, а не только exe |

## Повторный запуск (пересборка)

Просто дважды щёлкните по `START_WINDOWS.bat` снова.

Виртуальное окружение будет переиспользовано, новые зависимости доустановятся автоматически.

---

## PowerShell-вариант

Если вы предпочитаете PowerShell, используйте `START_WINDOWS.ps1` — он делает то же самое, но с цветным выводом и более подробными сообщениями.

Запуск из PowerShell:
```powershell
powershell -ExecutionPolicy Bypass -File START_WINDOWS.ps1
```

Или один раз разрешите запуск скриптов:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
.\START_WINDOWS.ps1
```

## Разница между START_WINDOWS.bat и build_exe.bat

| | `START_WINDOWS.bat` | `build_exe.bat` |
|---|---|---|
| Поиск Python | `py -3` + fallback `python` | только `python` |
| Создание venv | Да | Да |
| Проверка кода | Да (compileall + headless + тесты) | Нет |
| Сборка EXE | Да | Да |
| Предложение запустить | Да | Нет (только открыть папку) |
| Для кого | Конечный пользователь | Разработчик |
