# Сборка EXE — OzonFbsLabelPrinter

Полное руководство по сборке автономного `.exe`-файла для Windows.

---

## Почему нельзя собрать EXE из Linux?

PyInstaller работает по принципу «собирай там, где запускаешь»:

- Он упаковывает **работающий интерпретатор Python** и **нативные библиотеки** целевой ОС.
- PySide6 (Qt6) на Linux использует `.so`-библиотеки Linux, а на Windows — `.dll`-библиотеки Windows. Они **несовместимы**.
- Windows-специфичные модули (`win32api`, `win32print`) существуют только на Windows.
- Итоговый `.exe` — это Windows PE-бинарник; Linux не умеет его создать без эмуляции.

**Короткий ответ:** PyInstaller не поддерживает кросс-компиляцию. Собирать `.exe` нужно только на машине с Windows.

Альтернатива — использовать GitHub Actions с `windows-latest` runner или виртуальную машину Windows.

## Требования для сборки на Windows

| Компонент | Требование | Где взять |
|---|---|---|
| Windows | 10 или 11 (64-bit) | — |
| Python | **3.10, 3.11, 3.12 или 3.13 (64-bit, CPython)**. Рекомендуется 3.11 или 3.12. | https://python.org/downloads/ |
| pip | любая | идёт с Python |
| Интернет | — | для скачивания пакетов |

> **Важно:** устанавливайте **64-битный Python** с [python.org](https://www.python.org/downloads/windows/) — PySide6 не совместим с 32-bit Python, Python из Microsoft Store или слишком старыми версиями. При установке отметьте «Add Python to PATH».

## Ошибка «No matching distribution found for PySide6»

Если при установке зависимостей появляется:

```
ERROR: Could not find a version that satisfies the requirement PySide6>=6.5.0 (from versions: none)
```

**Причина:** Python несовместим с PySide6. Чаще всего это 32-bit Python, слишком старая версия или Python из Microsoft Store.

**Как исправить:**

1. Удалите папку `.venv` в папке проекта.
2. Установите **Python 3.11 или 3.12 (64-bit)** с [python.org](https://www.python.org/downloads/windows/).
3. При установке отметьте **«Add python.exe to PATH»**.
4. Запустите `START_WINDOWS.bat` снова.

Для диагностики запустите **`CHECK_PYTHON_WINDOWS.bat`** — он покажет все найденные Python и их совместимость с PySide6.

## Как собрать: быстрый старт

### Вариант 0 — Один клик (самое простое)

Дважды щёлкните по **`START_WINDOWS.bat`** — скрипт сам выполнит все шаги: найдёт Python,
создаст окружение, установит зависимости, проверит код, соберёт EXE и предложит запустить приложение.

Подробная инструкция: **[README_ONE_CLICK.md](README_ONE_CLICK.md)**

---

### Вариант A — Пакетный файл (рекомендуется)

1. Откройте **Командную строку** (Win+R → `cmd`).
2. Перейдите в папку проекта:
   ```bat
   cd C:\путь\к\ozon_fbs_label_app
   ```
3. Запустите сборку:
   ```bat
   build_exe.bat
   ```
4. Дождитесь завершения (1–3 минуты).
5. Результат в папке `dist\OzonFbsLabelPrinter\`.

### Вариант B — PowerShell

1. Откройте **PowerShell** в папке проекта.
2. Если скрипты отключены — разрешите для текущего пользователя:
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
   Или запустите в обход политики:
   ```powershell
   powershell -ExecutionPolicy Bypass -File build_exe.ps1
   ```
3. Запустите:
   ```powershell
   .\build_exe.ps1
   ```

### Вариант C — Вручную (шаг за шагом)

```bat
:: 1. Создать виртуальное окружение
python -m venv .venv

:: 2. Активировать
.venv\Scripts\activate

:: 3. Обновить pip
python -m pip install --upgrade pip

:: 4. Установить зависимости проекта
pip install -r requirements.txt

:: 5. Установить инструменты сборки
pip install pyinstaller pywin32

:: 6. (Опционально) Smoke-тест перед сборкой
smoke_test.bat

:: 7. Собрать EXE
pyinstaller --clean OzonFbsLabelPrinter.spec
```

## Где найти собранный EXE

После успешной сборки:

```
ozon_fbs_label_app\
└── dist\
    └── OzonFbsLabelPrinter\          ← скопируйте всю эту папку
        ├── OzonFbsLabelPrinter.exe   ← запускаемый файл
        ├── templates\                 ← шаблоны этикеток (включены автоматически)
        ├── _internal\                 ← рантайм Python и зависимости
        └── ...
```

> **Важно:** копируйте **всю папку** `OzonFbsLabelPrinter\`, а не только `.exe`. Приложение работает в режиме onedir — все файлы рядом с exe обязательны.

---

## Почему выбран режим onedir, а не onefile?

| Параметр | onefile | onedir (используется) |
|---|---|---|
| Запуск | Медленнее (распаковка во `%TEMP%`) | Быстро (без распаковки) |
| Антивирус | Часто блокирует | Реже блокирует |
| Отладка | Сложнее | Проще (файлы видны рядом) |
| Копирование | Один файл | Папка целиком |

## Настройка принтера Xprinter XP-365B

Xprinter XP-365B — термопринтер для этикеток 58 мм. Для корректной печати:

### 1. Установка драйвера

- Скачайте драйвер с [официального сайта Xprinter](https://www.xprintertech.com/download).
- Установите как обычный Windows-принтер.
- В диспетчере устройств убедитесь, что принтер отображается без ошибок.

### 2. Настройка размера бумаги

В свойствах принтера (Панель управления → Устройства и принтеры → ПКМ → Свойства принтера):
- Выберите или создайте формат бумаги **58 × 40 мм**.
- Установите его как размер по умолчанию.

### 3. Выбор принтера в приложении

- В выпадающем списке **«Принтер»** выберите «Xprinter XP-365B» (или аналогичное имя).
- Убедитесь, что в настройках шаблона установлены размеры **58 × 40 мм**.
- Нажмите **«Печать выбранных»** или **«Печать всех»**.

### 4. Если принтер не печатает

- Убедитесь что принтер онлайн (синий светодиод, нет «Offline» в списке принтеров).
- Попробуйте режим **«сохранить в файл»** и откройте PDF вручную — `output\print_jobs\`.
- При наличии `pywin32` будет использован `win32api.ShellExecute("printto")` — прямая отправка без диалогов.
- Без `pywin32` — `os.startfile("print")` откроет системный обработчик PDF.

## Если антивирус блокирует EXE или SmartScreen ругается

**Это нормально для самоподписанных/самосборных приложений.** PyInstaller-собранные файлы часто вызывают ложные срабатывания.

### SmartScreen (Windows)

При первом запуске появится синее окно «Windows защитила ваш компьютер»:

1. Нажмите **«Подробнее»** (ссылка внизу окна).
2. Нажмите **«Выполнить в любом случае»**.
3. Приложение запустится. Это предупреждение появляется только при первом запуске или после замены exe-файла.

### Антивирус (Kaspersky, ESET, Defender и др.)

1. Добавьте папку `dist\OzonFbsLabelPrinter\` в **исключения антивируса** перед запуском.
2. Если антивирус удалил файлы — восстановите из карантина и добавьте в исключения.
3. В крайнем случае — собирайте EXE заново после добавления папки в исключения.

**Почему так происходит:** PyInstaller использует технику упаковки, похожую на самораспаковывающиеся архивы. Эвристические анализаторы антивирусов иногда считают это подозрительным. Исходный код приложения открыт — вы можете убедиться в его безопасности самостоятельно.

## Часто задаваемые вопросы

**Q: EXE запускается, но не находит шаблоны (templates/)**

A: Скопируйте всю папку `dist\OzonFbsLabelPrinter\`, а не только `.exe`. Шаблоны включены в папку `templates\` рядом с exe. При первом запуске, если шаблонов нет, приложение создаст шаблон по умолчанию.

**Q: Долго запускается (10–15 секунд)**

A: Это нормально для onedir-сборок при первом запуске. Последующие запуски быстрее (файлы кэшируются ОС).

**Q: Ошибка "Failed to execute script run" при запуске**

A: Запустите `OzonFbsLabelPrinter.exe` из командной строки чтобы увидеть детальную ошибку:
```bat
cd dist\OzonFbsLabelPrinter
OzonFbsLabelPrinter.exe
```

**Q: Как собрать с иконкой приложения?**

A: Добавьте файл `assets\icon.ico` в папку проекта, затем раскомментируйте строку в `OzonFbsLabelPrinter.spec`:
```python
# icon="assets/icon.ico",   ← убрать # и пересобрать
```

## Автоматическая сборка через GitHub Actions

Если у вас нет Windows-машины или вы хотите автоматизировать сборку — используйте встроенный GitHub Actions workflow.

- Workflow запускается вручную, при push в `main`/`master` и при создании тега `v*`.
- На `windows-latest` раннере устанавливаются зависимости, выполняются проверки и собирается EXE.
- Результат доступен как артефакт Actions или как вложение GitHub Release (при теге).
- Секреты Ozon для сборки **не нужны** — используется mock/no-gui режим.

Подробная инструкция: **[README_GITHUB_ACTIONS.md](README_GITHUB_ACTIONS.md)**
