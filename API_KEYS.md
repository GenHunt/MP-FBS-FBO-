# 🔑 Управление API-ключами в приложении

Отличный вопрос! **Да, вы можете вводить API-ключи прямо в интерфейсе приложения** - это даже предпочтительный способ!

## ✅ 2 способа ввода API-ключей

### Способ 1: Прямо в интерфейсе приложения (РЕКОМЕНДУЕТСЯ) ⭐

Это самый удобный способ - ключи вводятся при каждом запуске приложения.

**Шаги:**

1. **Запустите приложение**
   ```bash
   python main.py
   # или запустите .exe файл
   Ozon-FBS-Label-Printer.exe
   ```

2. **Вы увидите главное окно с полями ввода**

   ```
   ┌─────────────────────────────────────────────────────────┐
   │ Ozon FBS Label Printer                                  │
   ├─────────────────────────────────────────────────────────┤
   │                                                           │
   │ Client ID:  [___________________________]                │
   │                                                           │
   │ API Key:    [___________________________] (скрыто)       │
   │                                                           │
   │ [✓ Подключиться]     ❌ Не подключено                   │
   │                                                           │
   └─────────────────────────────────────────────────────────┘
   ```

3. **Введите свои данные:**
   - **Client ID:** Скопируйте из личного кабинета Ozon
   - **API Key:** Скопируйте из личного кабинета Ozon

4. **Нажмите кнопку "✓ Подключиться"**

5. **При успехе увидите:**
   ```
   ✅ Подключено к Ozon
   ```

### Где получить API-ключи?

1. **Перейдите на Ozon Seller:** https://seller.ozon.ru/
2. **Откройте:** Личный кабинет → Интеграция → API
3. **Найдите:**
   - **Client ID** (обычно 5-6 цифр)
   - **API Key** (длинная строка символов)
4. **Скопируйте и вставьте в приложение**

## Способ 2: Через файл .env (Альтернативный)

Если вы предпочитаете сохранять ключи в файле (удобно для автоматизации).

**Шаги:**

1. **Создайте файл `.env` в корне проекта:**
   ```
   MP-FBS-FBO-/
   ├── .env          ← создайте этот файл
   ├── main.py
   └── ...
   ```

2. **Содержимое файла `.env`:**
   ```env
   OZON_CLIENT_ID=12345678
   OZON_API_KEY=your_very_long_api_key_here
   PRINTER_NAME=Xprinter XP-365B
   ```

3. **Замените на ваши реальные значения**

4. **При запуске приложение автоматически загрузит эти ключи**

## 🔄 Переключение между ключами

Если у вас несколько аккаунтов Ozon:

**Способ 1 (Интерфейс):**
- Просто введите другие Client ID и API Key
- Нажмите "✓ Подключиться"
- Готово!

**Способ 2 (Файл):**
- Отредактируйте `.env` файл
- Перезагрузите приложение

## 🛡️ Безопасность API-ключей

### ⚠️ Важные правила:

1. **Никогда не делитесь ключами**
   - ❌ Не отправляйте по email
   - ❌ Не выкладывайте на GitHub
   - ❌ Не показывайте на скриншотах

2. **Скройте ключи в `.env`**
   - Добавьте `.env` в `.gitignore`:
   ```
   .env
   .env.local
   *.key
   ```

3. **Используйте разные ключи для разных окружений**
   - Разработка: один ключ
   - Производство: другой ключ

4. **Регулярно ротируйте ключи**
   - Генерируйте новые ключи в личном кабинете Ozon
   - Удаляйте старые ключи

## 🔐 Защита .env файла

При распространении приложения:

### Для себя / своей команды:
- Можете оставить `.env` с ключами
- Убедитесь, что доступ ограничен

### Для клиентов / публичного дистрибутива:
- **НЕ включайте `.env` с ключами**
- Используйте `.env.example`:
  ```env
  OZON_CLIENT_ID=your_client_id_here
  OZON_API_KEY=your_api_key_here
  PRINTER_NAME=Xprinter XP-365B
  ```
- Каждый пользователь создает свой `.env`

OZON_API_KEY=sk-prod-abc123def456ghi789jkl...

Потом в интерфейсе или коде:
```python

# Хардкод прямо в коде - НИКОГДА ТАК НЕ ДЕЛАЙТЕ!
client = OzonAPIClient(
    client_id='87654321',
    api_key='sk-prod-abc123def456ghi789jkl...'
)
```

## 🚨 Если ключ скомпрометирован

1. **Немедленно отключите старый ключ**
   - В личном кабинете Ozon → Настройки API

2. **Создайте новый ключ**
   - Ozon автоматически сгенерирует новый

3. **Обновите ключ в приложении**
   - Через интерфейс или `.env`

4. **Проверьте логи**
   - Ищите в `logs/ozon_label_printer.log` подозрительную активность

## 📝 Варианты конфигурации

## 🔧 Как это работает в коде

В главном окне (`src/ui/main_window.py`):

```python
def connect_to_api(self) -> None:
    """Connect to Ozon API"""
    # Берет значения из текстовых полей интерфейса
    client_id = self.client_id_input.text().strip()
    api_key = self.api_key_input.text().strip()
    
    # Создает API клиент
    self.api_client = OzonAPIClient(client_id, api_key)
    
    # Проверяет подключение
    if self.api_client.test_connection():
        self.status_label.setText("✅ Подключено к Ozon")
        # Загружает заказы
        self.shipments_tab.load_shipments()
```

## ❓ FAQ

**Q: Где-то безопаснее хранить ключи?**
A: В `.env` файле на локальной машине (не в репо) или в переменных окружения OS.

**Q: Могу ли я использовать одни ключи для разработки и продакшена?**
A: Не рекомендуется! Используйте разные ключи для безопасности.

**Q: Как часто нужно менять ключи?**
A: Минимум раз в 3 месяца. Чаще, если подозреваете компрометацию.

**Q: Что делать если забыл ключ?**
A: Просто сгенерируйте новый в личном кабинете Ozon.
